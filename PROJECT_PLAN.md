# Project Plan: cd-headunit

Локальный план задач, синхронизированный с Master Plan.

## Стек (Фиксировано):

- **Backend:** FastAPI (Python)
- **Frontend:** React + Tailwind (SPA)
- **IPC:** MQTT / Local Socket (через cd-protocol)

## Текущие задачи:

### 1. Подготовка UI (React + FastAPI)

- [*] Инициализация React проекта в `src/frontend`
- [*] Инициализация FastAPI проекта в `src/backend`
- [*] Реализация базового API для диагностики системы
- [*] Реализация веб-интерфейса для внешнего подключения

### 2. OS Layer & Kiosk

- [ ] Исследование и настройка Kiosk Mode - _Отложено_
- [ ] Реализация кастомного загрузочного экрана (Splash Screen) - _Отложено_
- [ ] Интеграция UI в Kiosk режим

## Спецификации интерфейса (Design System):

- **Целевое разрешение:** 2880x824 (12.7", 3.5:1 aspect ratio).
- **Режим отладки:** Широкоэкранный формат центрируется по вертикали на стандартных мониторах.
- **Дизайн:** Темная премиальная тема (Deep Gray / Accent Red / Glassmorphism).
- **Сенсорный ввод:** Интерфейс оптимизирован под тач; отсутствие hover-эффектов и реакций на курсор.
- **Анимация:** Плавные переходы (горизонтальный аккордеон) 0.6s.

### 3. Функциональные модули

- [*] Разработка дизайн-системы (Tailwind config + Common Components)
- [*] Реализация главного дашборда (5-блочная симметричная архитектура)
- [*] Реализация детального вида с графиками (Dynamic Transitions)
- [*] Интеграция UI с системой диагностики (Simulated & Health API)
- [ ] Интерфейс управления OTA (ранняя стадия)
- [ ] Уведомления в UI о доступности новых пакетов в /data/incoming_updates

### 4. Инфраструктура

- [ ] Gateway-Service (перехват и маршрутизация пакетов)
- [ ] Сбор телеметрии с ESP-узлов в локальную БД
